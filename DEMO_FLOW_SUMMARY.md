# 데모 플로우 정리

## 기본 플로우
1. **데모 페이지**: `/demo` - 영업용 소개 페이지
2. **데모보기 버튼 클릭**: `/qr/demo_cafe_001`로 이동
3. **데모 데이터 표시**: 백엔드 DB의 데모 데이터를 그대로 표시

## 구현 방식
- 데모는 단순하게: 백엔드에 저장된 데모 데이터를 API로 가져와서 표시
- 복잡한 로컬 폴백이나 특별한 처리 없이 일반 API 호출과 동일하게 처리
- 데모 모드 구분은 QR ID가 `demo_`로 시작하는지로만 판단

## 현재 문제점
- 로컬 데이터와 API 데이터 충돌
- 복잡한 폴백 로직으로 인한 혼란
- 타입 불일치 문제

## 해결 방안
- 데모도 일반 API 호출과 동일하게 처리
- 백엔드에서 데모 데이터를 정확한 타입으로 제공
- 프론트엔드는 단순히 데이터를 표시만 함
