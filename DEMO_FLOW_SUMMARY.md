# SpotLine 데모 플로우 개선 요약 (최종 버전)

## 최종 개선된 데모 플로우

### 1. 완전히 분리된 플로우
```
홈페이지 → [🎭 데모 버튼] → /qr/demo_cafe_001 → /spotline/demo-store?qr=demo_cafe_001
```

### 2. 단계별 상세 플로우

#### Step 1: 홈페이지에서 데모 시작
- **위치**: 홈페이지(`/`) 하단 CTA 섹션
- **버튼**: "🎭 데모보기로 먼저 체험해보기"
- **동작**: `window.location.href = '/qr/demo_cafe_001'`
- **특징**: `/demo` 페이지 완전 제거, 홈에서 직접 시작

#### Step 2: QR 페이지에서 데모 감지
- **URL**: `/qr/demo_cafe_001`
- **감지 로직**: `qrId === "demo_cafe_001"`
- **동작**: `router.replace('/spotline/demo-store?qr=demo_cafe_001')`
- **특징**: 실제 QR 처리와 완전히 분리된 로직

#### Step 3: 데모 전용 SpotLine 페이지
- **URL**: `/spotline/demo-store?qr=demo_cafe_001`
- **데이터**: 하드코딩된 데모 데이터 (API 호출 없음)
- **특징**: 실제 운영 페이지(`/spotline/[qrId]`)와 완전 분리

## 핵심 개선사항

### 1. 완전한 라우트 분리
- ❌ **기존**: `/spotline/[qrId]`에서 데모 모드 플래그로 구분
- ✅ **개선**: `/spotline/demo-store` 전용 라우트로 완전 분리

### 2. 데이터 처리 방식 변경
- ❌ **기존**: 데모 API 호출 또는 복잡한 조건부 로직
- ✅ **개선**: 하드코딩된 데모 데이터, 로딩 시뮬레이션만 추가

### 3. 플로우 단순화
- ❌ **기존**: `/demo` 페이지 → 복잡한 데이터 처리 → SpotLine
- ✅ **개선**: 홈페이지 → QR 페이지 → 데모 전용 SpotLine

## 기술적 구현 상세

### 1. 라우트 구조
```
실제 운영:
/spotline/[qrId] → 동적 라우트, DB 조회, 통계 수집
예: /spotline/695f96c2825a4a7c28bb6ce9?qr=real_qr_001

데모 전용:
/spotline/demo-store → 정적 라우트, 하드코딩 데이터, 통계 없음
예: /spotline/demo-store?qr=demo_cafe_001
```

### 2. QR 처리 로직
```typescript
// src/app/qr/[qrId]/page.tsx
useEffect(() => {
  const handleQRScan = async () => {
    if (qrId === "demo_cafe_001") {
      // 데모 전용 라우트로 이동
      router.replace(`/spotline/demo-store?qr=${qrId}`);
      return;
    }
    
    // 실제 QR 처리
    const { storeId } = await getStoreIdByQR(qrId);
    router.replace(`/spotline/${storeId}?qr=${qrId}`);
  };
  
  handleQRScan();
}, [qrId, router]);
```
