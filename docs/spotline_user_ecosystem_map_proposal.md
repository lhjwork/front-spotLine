# SpotLine × 유저 생태계 지도 통합 설계 전략 제안서

## 배경

### 이전 전략 요약
- 콘텐츠 기반 플랫폼 전환 전략(content-based_transition_strategic_proposal.md)에서 "크루가 큐레이션한 장소 콘텐츠"를 중심으로 서비스 확장하는 방향을 제안
- 현재 Mockup A/B/C 방향에서 크루 큐레이션 콘텐츠를 중심으로 설계됨

### 새로운 설계 필요성
현재 설계에서는 **크루(SpotLine 운영자)만이 콘텐츠 생산자**입니다. 하지만 서비스가 성장하려면:

1. **유저 참여 기반 콘텐츠 확장**이 필요함 — 크루 인력만으로 콘텐츠 양을 확보하기 어려움
2. **지도 위에서 한눈에 보는 경험**이 필요함 — 사용자가 "이 동네에 뭐가 있지?"라는 질문에 바로 답을 얻을 수 있어야 함
3. **SpotLine 제휴의 가치가 시각적으로 드러나야 함** — 제휴 매장과 유저 추천 Spot이 공존하되 차별화되어야 함

### 핵심 아이디어
> **지도 위에 SpotLine 제휴 Spot + 유저들이 방문·좋아요·소개한 Spot이 함께 표시되는 생태계**

#### 지도 = 하나의 캔버스, 두 개의 데이터 소스

```
┌──────────────────────────────────────────────────────────────────┐
│                        🗺️  지도 (캔버스)                         │
│                                                                  │
│   ┌───────────────────────────┐  ┌────────────────────────────┐  │
│   │  ⚡ SpotLine 제휴 Spot     │  │  👤 유저 활동 Spot          │  │
│   │                           │  │                            │  │
│   │  각 SpotLine별로           │  │  회원가입 유저가            │  │
│   │  크루가 큐레이션한         │  │  자발적으로 활동한          │  │
│   │  제휴 매장들               │  │  Spot들                    │  │
│   │                           │  │                            │  │
│   │  • 성수 크리에이티브 라인  │  │  📍 방문 "여기 다녀왔어요"  │  │
│   │    → 온도커피, 대림창고    │  │  ❤️ 좋아요 "이 Spot 좋아"  │  │
│   │  • 을지로 레트로 라인      │  │  📢 소개 "이 Spot 추천!"   │  │
│   │    → 미도식당, 세운상가    │  │                            │  │
│   │  • 연남 감성 산책 라인     │  │  → 유저 활동이 쌓이면       │  │
│   │    → 카페레이어드, ...     │  │    해당 Spot에 Social Proof │  │
│   └───────────────────────────┘  └────────────────────────────┘  │
│                                                                  │
│   지도 위에서 두 데이터가 마커 형태로 공존                         │
│   → SpotLine 마커: ⚡ 컬러 테두리 + SpotLine 이름                 │
│   → 유저 활동 마커: 👤 아바타 + 방문·좋아요·소개 카운트            │
│                                                                  │
│   ═══════════════════════════════════════════                     │
│   SpotLine 제휴 Spot에도 유저 활동이 누적됨                       │
│   → "이 제휴 매장을 132명이 방문, 67명이 좋아요"                  │
│   → 유저 활동 = SpotLine 제휴의 가치를 증명하는 지표              │
│   ═══════════════════════════════════════════                     │
└──────────────────────────────────────────────────────────────────┘
```

#### 유저 입장에서 보이는 지도 경험

```
유저가 지도를 열면:

1. 내 위치 주변에 마커들이 보인다
   ├─ ⚡ SpotLine 제휴 마커 ("성수라인의 온도커피")
   │   → 크루가 큐레이션한 공식 제휴 매장
   │   → 해당 SpotLine의 컬러로 표시
   │   → 다른 유저들의 방문/좋아요 수 함께 표시
   │
   └─ 👤 유저 활동 마커 ("커피탐험가님이 소개한 골목끝카페")
       → 회원 유저가 방문/좋아요/소개한 Spot
       → 추천자의 아바타와 한줄 후기 표시
       → 다른 유저들의 활동이 추가로 쌓임

2. 마커를 탭하면:
   ├─ SpotLine Spot → 크루 스토리 + 매장 정보 + 유저 활동 집계
   └─ 유저 Spot    → 유저 후기 + 방문자 목록 + 좋아요 수

3. 필터를 바꾸면:
   ├─ [전체]        → 모든 마커 표시
   ├─ [SpotLine만]  → 제휴 Spot만 (SpotLine별 컬러 구분)
   ├─ [유저 활동만] → 유저가 방문/좋아요/소개한 Spot만
   └─ [카테고리]    → 카페, 맛집, 전시 등
```

## 현재 Mockup 분석

### 현재 구조의 한계

| 구분 | 현재 상태 | 문제점 |
|------|----------|--------|
| Spot 소스 | 크루만 등록 | 콘텐츠 양 확보 어려움, 성장 한계 |
| 유저 역할 | 없음 (조회만) | 참여/재방문 동기 부족 |
| 지도 표시 | 모든 Spot이 동일하게 표시 | SpotLine 제휴의 가치가 안 보임 |
| 소셜 요소 | 없음 | 방문 기록, 좋아요, 추천 등 사회적 증거(Social Proof) 부재 |
| 방문 데이터 | 없음 | 실제 인기도/활성도 파악 불가 |

### 축 1: SpotLine 제휴 Spot (공급측 — B2B)

크루가 큐레이션하고, 매장이 공식 제휴한 Spot. **각 SpotLine별로 고유 컬러를 가지며, 지도 위에서 SpotLine 단위로 그룹화되어 표시**됩니다.

#### 지도 위 표현 방식

```
지도 위에서 SpotLine 제휴 Spot은 이렇게 보입니다:

  🔵 성수 크리에이티브 라인            🟡 을지로 레트로 라인
  ┌─────────────────┐          ┌─────────────────┐
  │ ⚡ 온도 커피       │          │ ⚡ 미도식당       │
  │ 파란색 테두리 마커 │          │ 노란색 테두리 마커 │
  │ ✔️ 파트너 뼉지     │          │ ✔️ 파트너 뼉지     │
  │ 크루: 민지         │          │ 크루: 지훈         │
  │                 │          │                 │
  │ 📍132 ❤️67 📢12  │          │ 📍89  ❤️45 📢8   │
  │ (유저 활동 집계)   │          │ (유저 활동 집계)   │
  └─────────────────┘          └─────────────────┘

  → SpotLine별 컬러로 구분 (파란, 노란, 보라 등)
  → ⚡ 아이콘으로 SpotLine 제휴 Spot임을 표시
  → 유저 활동 집계(방문수/좋아요/소개)가 마커에 함께 표시
```

| 속성 | 설명 |
|------|------|
| 등록 주체 | 크루 (SpotLine 운영자) |
| 매장 제휴 | 매장과 파트너십 계약 (QR 코드, 분석 제공) |
| 콘텐츠 | 크루의 큐레이션 스토리 + 매장 정보 |
| 지도 표시 | **SpotLine 브랜딩 마커** (SpotLine 이름 + 컬러) |
| 수익 모델 | 매장 → SpotLine (제휴비, 분석 리포트) |
| 신뢰도 | 크루 검증 + 매장 공식 승인 = 높은 신뢰 |
| 차별화 요소 | QR 코드 스캔 → 현장 경험 연결, "다음 Spot 제안" |

### 축 2: 유저 활동 Spot (수요측 — UGC)

회원가입한 유저가 직접 발견하고 방문·좋아요·소개하는 Spot. **유저의 실제 활동이 지도에 반영되어 살아있는 지도를 만듭니다.**

#### 지도 위 표현 방식

```
지도 위에서 유저 활동 Spot은 이렇게 보입니다:

  유저가 소개한 Spot               유저 활동만 있는 Spot
  (새로운 Spot 등록)            (기존 Spot에 활동 누적)
  ┌─────────────────┐          ┌─────────────────┐
  │ 👤 골목끝 카페     │          │ ⚡ 온도커피       │
  │ 보라색 핀 마커     │          │ + 유저 활동 레이어 │
  │                 │          │                 │
  │ 소개: 커피탐험가    │          │ 커피탐험가님     │
  │ "숨은 명카페!"    │          │ 방문 + 좋아요     │
  │ 📍23  ❤️15  📢5  │          │ 맛집헌터님       │
  │                 │          │ 방문 + 소개        │
  └─────────────────┘          └─────────────────┘

  → 유저 소개 Spot: 보라색/회색 핀 + 소개자 아바타
  → 유저 활동이 많을수록 마커 크기가 커짐 (인기도 시각화)
  → 최근 방문자들의 아바타가 마커 주변에 작게 표시
```

| 속성 | 설명 |
|------|------|
| 등록 주체 | 회원가입 유저 |
| 매장 제휴 | 없음 (유저 자발적 소개) |
| 콘텐츠 | 유저의 방문 후기, 사진, 짧은 코멘트 |
| 지도 표시 | **유저 마커** (일반 핀 + 추천자 아바타) |
| 수익 모델 | 인기 유저 Spot → SpotLine 제휴 전환 리드 |
| 신뢰도 | 유저 평판 기반 (방문 수, 추천 횟수) |
| 차별화 요소 | 자연스러운 콘텐츠 확장, 크루 인력 의존 탈피 |

### 축 3: 유저 활동 레이어 (양쪽 Spot 모두에 적용)

회원 유저의 방문·좋아요·소개 활동이 **SpotLine 제휴 Spot과 유저 소개 Spot 모두에** 누적됩니다.

#### 지도 위에서 유저 활동이 반영되는 방식

```
SpotLine 제휴 Spot "온도커피"에 유저 활동이 쌓이는 과정:

  커피탐험가님  →  "여기 다녀왔어요" (Visit)   → 방문 카운트 +1
  맛집헌터님  →  "이 Spot 좋아요" (Like)    → 좋아요 카운트 +1
  도시산책러님  →  "이 Spot 추천!" (Recommend) → 소개 카운트 +1 + 후기

  지도 마커에 반영:
  ┌───────────────────────────────┐
  │  ⚡ 온도커피 (파란 테두리)       │
  │  성수 크리에이티브 라인            │
  │                               │
  │  📍 132명 방문  ·  ❤️ 67개  ·  📢 12건 │
  │                               │
  │  최근 방문자:                   │
  │  👤👤👤👤👤 +127명              │
  └───────────────────────────────┘

  → SpotLine 제휴 Spot에 유저 활동이 누적 = Social Proof
  → 제휴 매장 입장에서 "132명이 방문했다" = 제휴 가치 증명
  → 유저 소개 Spot에도 동일하게 적용
```

| 활동 타입 | 설명 | 지도 반영 |
|-----------|------|-----------|
| **방문 (Visit)** | "여기 다녀왔어요" | 방문 수 뱃지, 최근 방문자 표시 |
| **좋아요 (Like)** | "이 Spot이 좋아요" | 인기 순위에 반영, 하트 카운트 |
| **소개 (Recommend)** | "이 Spot을 소개합니다" + 후기 | 유저 추천 Spot으로 등록 가능 |

### 유저 활동별 지도 표현 상세

회원 유저의 3가지 활동(방문, 좋아요, 소개)이 지도에서 어떻게 표현되는지:

#### 방문 (Visit) — "여기 다녀왔어요"

```
지도 마커에 반영:                      Spot 상세 페이지에서:
┌─────────────────────┐          ┌──────────────────────────────┐
│ 📍 132명 방문         │          │  최근 방문자                        │
│ 👤👤👤👤👤 +127          │          │                              │
│ (최근 방문자 아바타)    │          │  👤 커피탐험가 • 2시간 전          │
└─────────────────────┘          │  "역시 핸드드립이 최고!"        │
                                 │                              │
• 방문 카운트 버블              │  👤 도시산책러 • 어제            │
• 최근 방문자 아바타 나열      │  "조용한 분위기 좋아요"          │
• 방문 많을수록 버블 크기 증가  │                              │
                                 │  👤 맛집헌터 • 3일 전            │
                                 │  "커피와 케이크 두 다 맛있음"    │
                                 └──────────────────────────────┘
```

#### 좋아요 (Like) — "이 Spot이 좋아요"

```
지도 마커에 반영:                      Spot 인기도 시스템:
┌─────────────────────┐          ┌──────────────────────────────┐
│ ❤️ 67개 좋아요         │          │  좋아요 수에 따른 마커 크기:     │
│ ❤️ 마커에 하트 아이콘   │          │                              │
└─────────────────────┘          │  0~10개:  ● 기본 크기 마커       │
                                 │  10~50개: ● 중간 크기 마커        │
• 좋아요 수가 마커에 표시       │  50개+:  ● 큰 크기 + 보라 후광     │
• 좋아요 많을수록            │                              │
  마커가 커지고 후광 효과       │  → 인기 Spot이 지도에서 눈에 띔 │
• 인기 순위 필터 시 반영     │  → 필터로 "인기순" 정렬 가능     │
                                 └──────────────────────────────┘
```

#### 소개 (Recommend) — "이 Spot을 소개합니다"

```
새로운 Spot 등록 플로우:             지도 반영:
┌────────────────────────┐          ┌──────────────────────────────┐
│ 유저가 Spot 소개:         │          │  새 👤 마커가 지도에 등장:       │
│                        │          │                              │
│ 1. Spot 이름 입력       │          │  👤 골목끝카페                │
│ 2. 위치 선택 (지도)     │          │  소개: 커피탐험가              │
│ 3. 사진 1~3장 첨부     │          │  "숨은 명카페!"                │
│ 4. 한줄 후기 작성       │          │  📍 5명 방문 • ❤️ 3개            │
│ 5. 카테고리 선택        │          │                              │
│                        │          │  → 소개자의 아바타와 한줄 후기  │
│ → 지도에 새 마커 등장  │          │    가 마커에 함께 표시         │
└────────────────────────┘          │  → 다른 유저들의 활동 유도    │
                                 └──────────────────────────────┘
```

### 지도 필터 옵션

| 필터 그룹 | 필터 | 설명 |
|-----------|------|------|
| **소스 필터** | 전체 보기 | SpotLine 제휴 + 유저 활동 Spot 모두 표시 |
| | SpotLine만 | 제휴 Spot만 표시 (SpotLine별 컬러 구분) |
| | 유저 활동만 | 유저가 방문/좋아요/소개한 Spot만 표시 |
| **SpotLine 필터** | 개별 SpotLine | "성수 크리에이티브 라인"만, "을지로 레트로 라인"만 등 |
| **유저 활동 필터** | 방문이 많은 순 | 방문 카운트 기준 정렬 |
| | 좋아요 많은 순 | 좋아요 카운트 기준 정렬 |
| | 최근 활동순 | 가장 최근에 활동이 있는 Spot 우선 |
| **카테고리 필터** | 카페, 맛집, 전시, 쇼핑, 산책 | 카테고리별 필터 |
| **인기순** | 종합 점수 | 방문 + 좋아요 + 소개 종합 점수 기반 |

#### 필터 조합 예시

```
유저가 "성수 크리에이티브 라인"을 선택하면:

  지도에 해당 SpotLine의 제휴 Spot만 표시
  + 각 Spot에 누적된 유저 활동(방문/좋아요/소개) 함께 표시

  ⚡온도커피      ⚡대림창고      ⚡할아버지공장
  📍132 ❤️67      📍89 ❤️45       📍67 ❤️23
  (모두 파란색 테두리 — 성수라인 컬러)

유저가 "유저 활동만" + "방문이 많은 순"을 선택하면:

  유저들이 방문한 Spot만, 방문 수 내림차순으로 표시
  SpotLine 제휴 Spot도 포함 (유저 방문 데이터가 있으면)

  ⚡온도커피(📍132)  👤골목끝카페(📍45)  ⚡미도식당(📍89)
  두 소스가 섬여서 표시, 색상으로 구분
```

### 상호작용 흐름 — 유저가 지도에서 경험하는 여정

```
1. 지도 진입 (비회원도 가능)
   └→ 현위치 중심으로 SpotLine 제휴 Spot + 유저 활동 Spot이 함께 표시
      색상/마커 형태로 두 소스가 자연스럽게 구분됨

2. SpotLine 제휴 Spot 마커 탭
   └→ 프리뷰 카드:
      • SpotLine 이름 ("성수 크리에이티브 라인")
      • 매장 이미지 + 크루 이름
      • 유저 활동 집계: 📍132명 방문 • ❤️67개 • 📢12건
      └→ 상세 페이지:
         • 크루의 큐레이션 스토리
         • 매장 정보 (영업시간, 주소, 연락처)
         • 유저 활동 섹션: 최근 방문자, 좋아요 수, 후기 목록
         • QR 스캔 안내 / "다음 Spot" 추천

3. 유저 활동 Spot 마커 탭
   └→ 프리뷰 카드:
      • 소개자 닉네임 + 아바타 ("커피탐험가님이 소개한 Spot")
      • 사진 + 한줄 후기
      • 유저 활동 집계: 📍23명 방문 • ❤️15개 • 📢5건
      └→ 상세 페이지:
         • 소개자의 후기 전문
         • 다른 방문자들의 후기/사진
         • 방문자 목록 + 좋아요 수
         • "나도 다녀왔어요" / "좋아요" 버튼 (회원만)

4. 유저 활동 수행 (회원 전용)
   ├→ "다녀왔어요" → 방문 기록 + 선택적 한줄 후기/사진
   │   → 해당 Spot 마커의 방문 카운트 +1
   │   → 상세 페이지 최근 방문자에 추가
   │
   ├→ "좋아요" → Spot에 좋아요 +1
   │   → 마커의 좋아요 카운트 증가
   │   → 인기 Spot일수록 마커 크기 증가
   │
   └→ "Spot 소개" → 새로운 유저 Spot 등록
       → 지도에 새 👤 마커 등장
       → 다른 유저들의 방문/좋아요 유도

5. 실시간 활동 피드 (지도 하단)
   └→ "커피탐험가님이 온도 커피를 방문했어요" (실시간 느낌)
      → 피드 탭 시 해당 Spot으로 지도 이동
```

### 방향 A: 지도 기반 통합 탐색

> **메타포:** "네이버 지도 위에 인스타그램을 얹은 것"
> **한줄:** 지도를 열면 SpotLine 제휴 Spot과 유저 추천 Spot이 다른 색상/형태로 함께 보인다.

#### 핵심 개념

```
┌────────────────────────────────────────┐
│              🗺️ 지도 뷰                │
│                                        │
│    ⚡ SpotLine 제휴 마커 (컬러 테두리)   │
│    👤 유저 추천 마커 (아바타 핀)         │
│    🔥 인기 Spot 강조 (크기 변동)        │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ [전체] [SpotLine만] [유저만]      │  │
│  │ [카페] [맛집] [전시] [산책]       │  │
│  └──────────────────────────────────┘  │
│                                        │
│  📋 리스트 전환 시:                     │
│  SpotLine 뱃지 + 유저 활동 카운트       │
│  + 최근 방문자 아바타 + 소셜 피드       │
└────────────────────────────────────────┘
```

#### 상세 설계

| 구분 | 설명 |
|------|------|
| 진입 | "시작하기" → 위치 허용 → 지도 중심 탐색 |
| 지도 마커 | SpotLine 제휴: 해당 SpotLine 컬러 테두리 + ⚡ 아이콘, 파트너는 ✓ 뱃지 |
| | 유저 추천: 보라색/회색 핀 + 추천자 아바타, 좋아요 수에 따라 크기 변동 |
| 리스트 뷰 | 소스 필터(전체/SpotLine/유저) + 카테고리 필터 |
| Spot 카드 | SpotLine 뱃지 + 방문/좋아요/추천 카운트 + 최근 방문자 아바타 |
| 소셜 피드 | "최근 유저 활동" 접이식 피드 (방문, 좋아요, 소개 실시간) |

### 방향 D: 스토리/릴스형 탐색 ⭐ NEW

| 장점 | 단점 |

| ✅ 몰입감 최고 — 풀스크린으로 Spot 매력 극대화 | ❌ 텍스트 정보 전달이 제한적 |

| ✅ SpotLine 스토리를 감성적으로 전달 가능 | ❌ 구현 복잡도 높음 (애니메이션, 제스처) |
| ✅ 중독성 있는 탐색 경험 | ❌ 초기에 충분한 비주얼 콘텐츠 필요 |

| 미션 유형 | 방문 미션 (N곳 방문), 루트 완주 미션, 리뷰 미션, SpotLine 미션 |
| 뱃지 시스템 | 미션 완료 시 뱃지 획득 → 프로필에 표시 → 다른 유저에게 보임 |
| 레벨 시스템 | 활동 포인트 누적 → 레벨업 (탐방 초보 → 도시 탐험가 → 로컬 마스터) |
| 리더보드 | 주간/월간 방문왕, 리뷰왕, 완주왕 랭킹 |
| SpotLine 연결 | "성수 크리에이티브 라인 완주" 같은 SpotLine 전용 미션 |
| 유저 추천 연결 | "첫 Spot 소개" → "인기 추천자" → "큐레이터 후보" 단계적 성장 |
| 소셜 공유 | 뱃지 획득, 루트 완주 시 소셜 공유 카드 자동 생성 |

#### 미션 예시

| 미션 | 조건 | 보상 |
|------|------|------|
| 첫 발자국 | 아무 Spot 1곳 방문 | 🏅 탐방 시작 뱃지 |
| 카페 헌터 | 카페 5곳 방문 | 🏅 카페 헌터 뱃지 |
| 성수 마스터 | 성수 지역 8곳 방문 | 🏅 성수 마스터 뱃지 |
| 루트 러너 | SpotLine 루트 1개 완주 | 🏅 루트 마스터 뱃지 |
| 리뷰 스타 | 한줄 후기 10개 작성 | 🏅 리뷰어 뱃지 |
| 추천왕 | Spot 3곳 소개 | 🏅 큐레이터 후보 뱃지 |
| 주간 챌린저 | 이번 주 5곳 이상 방문 | 주간 리더보드 등록 |

#### 장단점

| 장점 | 단점 |
|------|------|
| ✅ 재방문율 극대화 — 미션 완료 동기 | ❌ 구현 복잡도 높음 (미션/뱃지/레벨/리더보드) |
| ✅ 유저 활동 폭발적 증가 (방문, 리뷰, 소개) | ❌ 어뷰징 방지 필요 (GPS 조작 등) |
| ✅ 바이럴 효과 — 뱃지/랭킹 소셜 공유 | ❌ 게임 요소가 서비스 본질을 흐릴 수 있음 |
| ✅ SpotLine 루트 완주라는 명확한 목표 | ❌ 미션 설계/밸런싱에 지속적 관리 필요 |
| ✅ 매장에 실제 방문을 유도하는 효과 | ❌ 게이미피케이션에 반감을 가진 유저층 존재 |

#### 추천 조합: **F + E** (게이미피케이션 + 루트) 또는 **F + A** (게이미피케이션 + 지도)

### 시나리오별 추천 조합 — 상세 분석

#### 🟢 조합 1: MVP (최소 구현) — C 단독

> **"기존 QR 플로우는 건드리지 않고, 소셜 피드 하나만 추가하여 최소한의 유저 참여를 테스트"**

```
현재 서비스 ──── + ────── 커뮤니티 피드 (소셜)
     │                        │
  QR 스캔                  "둘러보기"
  기존 그대로               방문·좋아요·소개
                           활동 타임라인
```

| 항목 | 내용 |
|------|------|
| **적합한 시점** | 지금 당장 / 유저 반응을 먼저 확인하고 싶을 때 |
| **구현 범위** | 기존 랜딩에 "둘러보기" 버튼 + 피드 페이지 1개 추가 |
| **예상 공수** | 프론트 1~2주 (백엔드 피드 API 필요 시 +1주) |
| **검증 가능한 것** | "유저가 다른 사람의 활동을 보고 Spot에 관심을 갖는가?" |
| **리스크** | 매우 낮음 — 기존 코드 변경 없음, 실패해도 제거 쉬움 |
| **다음 단계** | 피드 반응이 좋으면 → A(지도) 또는 D(릴스) 추가 |

**이 조합을 선택해야 하는 경우:**
- 아직 어떤 방향이 맞는지 확신이 없을 때
- 개발 리소스가 제한적일 때
- "유저 참여"라는 개념 자체를 먼저 시장 테스트하고 싶을 때

---

#### 🔵 조합 2: 지도 중심 서비스 — A + E

> **"지도를 열면 SpotLine 제휴/유저 추천 Spot이 보이고, 루트(코스)를 따라 Spot을 순서대로 탐험"**

```
┌──────────────────────────────────────────┐
│              🗺️ 지도 (A)                  │
│                                          │
│   ⚡ SpotLine 마커     👤 유저 마커       │
│   ━━━━━━━ 루트 경로선 (E) ━━━━━━━        │
│   ① 온도커피 ──→ ② 할아버지공장 ──→ ③    │
│                                          │
│   [루트 선택] [SpotLine 필터] [유저 필터]  │
└──────────────────────────────────────────┘

유저 플로우:
1. 지도 열기 → 주변 Spot 한눈에 파악
2. "성수 카페 3곳 코스" 루트 선택
3. 지도 위에 경로선 + 순번 마커 표시
4. 첫 번째 Spot 방문 → "도착 인증" → 다음 Spot으로
5. 전체 완주 → 완주 뱃지 + 소셜 공유
```

| 항목 | 내용 |
|------|------|
| **적합한 시점** | SpotLine의 "다음 Spot 제안"을 핵심 경험으로 밀고 싶을 때 |
| **구현 범위** | 지도 뷰(A) + 루트 목록/상세(E) + 마커 차별화 + 경로선 |
| **예상 공수** | 프론트 3~4주 (지도 API 연동 + 루트 UI + 도착 인증) |
| **핵심 가치** | "지도 = 탐색" + "루트 = 목표" → 탐색과 목표가 동시에 존재 |
| **매장 영업 활용** | "이 루트에 포함되면 유저들이 순서대로 방문합니다" |
| **SEO 효과** | 높음 — "/routes/성수-카페-코스" 같은 롱테일 키워드 가능 |

**시너지 포인트:**
- 지도(A)가 **"어디에 뭐가 있지?"** 에 답하고
- 루트(E)가 **"그래서 어떤 순서로 갈까?"** 에 답함
- SpotLine의 "다음 Spot" 컨셉이 지도 위 경로선으로 시각화됨
- 루트 완주라는 명확한 목표가 재방문 동기 부여

**이 조합을 선택해야 하는 경우:**
- SpotLine = "동네 탐방 루트"로 포지셔닝하고 싶을 때
- 지도 기반 서비스로 차별화하고 싶을 때
- 오프라인 방문 유도가 핵심 목표일 때

#### 🟣 조합 3: 콘텐츠 성장 중심 — D + C

> **"릴스처럼 Spot을 소비하고, 소셜 피드로 활동이 확산되어 콘텐츠가 자연 성장"**

```
┌─────────────────────────────────────────┐
│  📱 릴스 탐색 (D)      📋 소셜 피드 (C) │
│                                         │
│  ┌──────────────┐   "커피탐험가님이      │
│  │ [풀스크린    │    온도 커피를 방문"    │
│  │  Spot 사진]  │                        │
│  │              │   "맛집헌터님이         │
│  │  온도 커피   │    을지로 루프탑을      │
│  │  ⚡ 성수라인  │    소개"               │
│  │  ❤️ 34      │                        │
│  │              │   → 피드에서 Spot 탭   │
│  │  ↕️ 스와이프 │   → 릴스 카드로 이동   │
│  └──────────────┘                        │
└─────────────────────────────────────────┘

콘텐츠 성장 루프:
유저가 Spot 방문 → 사진+한줄 올림 (D에 노출)
  → 다른 유저가 릴스로 발견 → 좋아요/방문
    → 활동이 소셜 피드(C)에 노출
      → 또 다른 유저가 관심 → 방문 → 콘텐츠 생산
        → 🔄 선순환
```

| 항목 | 내용 |
|------|------|
| **적합한 시점** | 크루 인력 의존을 벗어나 UGC로 성장하고 싶을 때 |
| **구현 범위** | 릴스형 카드 UI(D) + 소셜 타임라인(C) + 유저 콘텐츠 업로드 |
| **예상 공수** | 프론트 3~4주 (스와이프 UI + 피드 + 업로드 기능) |
| **핵심 가치** | 콘텐츠 소비(릴스)와 생산(후기)이 가장 낮은 허들로 연결 |
| **콘텐츠 확장** | 사진 1장 + 한줄 = 유저 진입 장벽 최소 |
| **바이럴 효과** | 릴스 카드 자체가 소셜 공유에 최적화된 포맷 |

**시너지 포인트:**
- 릴스(D)가 **"보기 좋은 발견"** 을 제공하고
- 소셜 피드(C)가 **"다른 사람도 가고 있다"** 는 증거를 제공
- 유저 콘텐츠 생산 허들이 가장 낮음 (사진 + 한줄)
- SpotLine 제휴 Spot은 릴스에서 ⚡ 뱃지로 프리미엄 표시

**이 조합을 선택해야 하는 경우:**

## 경쟁 서비스 대비 차별점

| 서비스 | 특징 | SpotLine과의 차이 |
|--------|------|-------------------|
| 네이버 지도 | 범용 지도 + 리뷰 | SpotLine은 "큐레이션 루트" 기반 |
| 카카오맵 | 범용 지도 + 즐겨찾기 | SpotLine은 "다음 Spot 제안" |
| 망고플레이트 | 맛집 전문 리뷰 | SpotLine은 카페+전시+쇼핑+산책 통합 |
| 식신 | AI 맛집 추천 | SpotLine은 사람(크루)의 큐레이션 |
| 트리플 | 여행 코스 추천 | SpotLine은 일상적 동네 탐방 |

**SpotLine 고유 가치:**
1. 크루의 **큐레이션 스토리** — 단순 정보가 아닌 맥락과 감성
2. **SpotLine(루트)** 단위의 연결 — "여기 다음은 여기" 흐름
3. **QR 현장 연결** — 매장에서 시작하는 오프라인-온라인 연결
4. **유저 활동의 시각화** — 방문·좋아요·소개가 지도에 실시간 반영

## 요약

### 생태계 설계 변화

| 현재 | 제안 |
|------|------|
| 크루만 콘텐츠 생산 | 크루 + 회원 유저가 함께 생태계 구성 |
| 지도에 동일한 핀 | SpotLine 제휴 마커(⚡컬러) / 유저 활동 마커(👤) 차별화 |
| 유저 참여 없음 | 방문·좋아요·소개 — 3가지 유저 활동이 지도에 반영 |
| 단일 데이터 소스 | 레이어 1: SpotLine 제휴 + 레이어 2: 유저 활동 (이중 소스) |
| 인기도 측정 불가 | 유저 활동(방문수/좋아요수) → 마커 크기로 인기도 시각화 |
| 성장 한계 (크루 인력) | 유저 소개 Spot → 인기 상승 → SpotLine 제휴 전환 파이프라인 |
| 3가지 Mockup 방향 | **7가지 방향 + 조합 가능** |

### 지도 위 핵심 설계 요약

```
🗺️ 지도 = 하나의 캔버스

  ⚡ SpotLine 제휴 마커              👤 유저 활동 마커
  ─────────────────────            ─────────────────────
  각 SpotLine별 고유 컬러            회색/보라 핀
  크루 큐레이션 + 매장 제휴           유저가 소개한 Spot
  ✓ 파트너 뱃지                     소개자 아바타 표시

  ═══════════ 양쪽 모두에 적용 ═══════════
  📍 방문 카운트  ❤️ 좋아요 수  📢 소개 수
  → 유저 활동이 마커에 실시간 반영
  → 활동 많을수록 마커 크기 증가 (인기도 시각화)
  → 최근 방문자 아바타가 마커 주변에 표시
```

### 7가지 방향 한줄 요약

| 방향 | 한줄 요약 | 핵심 키워드 |
|------|----------|------------|
| **A** 지도 통합 | 지도 위에 SpotLine+유저 Spot 구분 표시 | 🗺️ 공간 탐색 |
| **B** SpotLine 큐레이션 | SpotLine 단위 탐색 + 유저 인기 랭킹 | 📋 브랜드 큐레이션 |
| **C** QR + 소셜 | 기존 QR 유지 + 커뮤니티 피드 | 🔄 점진적 전환 |
| **D** 릴스형 | 풀스크린 스와이프로 Spot 탐색 | 📱 모바일 몰입 |
| **E** 루트 중심 | 코스 단위로 Spot 연결 + 완주 | 🚶 "다음 Spot" 극대화 |
| **F** 게이미피케이션 | 미션/뱃지/레벨/리더보드 | 🎮 재방문 중독성 |
| **G** AI 추천 | 취향 분석 + 개인 맞춤 추천 | 🤖 개인화 경험 |

> **핵심 메시지:** 지도 위에 SpotLine이 제휴한 Spot들과, 회원가입한 유저들이 방문·좋아요·소개하는 Spot이 함께 표시되며, 유저 활동이 양쪽 Spot 모두에 누적되어 살아있는 생태계 지도를 구축합니다. 7가지 방향 중 가장 적합한 조합을 선택하여 SpotLine만의 독자적인 탐색 경험을 만듭니다.
